工程思想经验：这些应该在刚开始就确定下来，确定好当时最佳的路线
1.用c写项目的时候，对于要输入的应该放在函数里面，这样才可以进行输入的错误检测。
或者更好的方法就是写一个函数对输入的东西进行检测，这样原本的函数还可以写参数，可以应用到其他的地方，专门性减少
2.对于求长度的函数，应该有返回值，不能在函数里面进行输出说长度为多少，否则其他的函数不好调用
3.在写函数的时候应该考虑这个函数是否会被调用，会的话就要写返回值，
是否要输入参数求其他的，是的话就要写参数，在函数外进行输入，




疑问：
void destroy(struct Node * phead) {
    struct Node* pt = phead;
    struct Node* temp = pt;
    while (pt != NULL) {
        temp = pt->next;
        free(pt);
        pt = temp;
    }
    printf("链表销毁完成");
    head = NULL;//易错，这里要把参数改为phead,防止全局变量和局部变量混淆
    tail = NULL;
}为什么这个不是Node ** head，但是却确实能修改实际的链表，有些就没有用.
解答：就算这个是指针类型也要，这里应该是因为free的原因，看看奇偶改变的那里就要这么用 参数用(Node** head),记得使用的时候用(&head)，别少了&





缺点：1.应该应用泛式函数，这样就可以少写很多重复的东西了
2.估计之前忘记搞释放空间，所以这个文件越来越大
